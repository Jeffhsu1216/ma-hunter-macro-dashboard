## 二、結帳週期與當期判定規則

**半年結帳制：每年 3 月底 / 9 月底結帳**

| 報告月份 | 本期起算日 | 本期結束日 |
|---------|----------|----------|
| 1～3 月 | 去年 10/1 | 今年 3/31 |
| 4～9 月 | 今年 4/1 | 今年 9/30 |
| 10～12 月 | 今年 10/1 | 明年 3/31 |

**當期交易篩選規則：**
- 僅計入**買入日期（M 欄）≥ 本期起算日**的交易
- 若該客戶為第一期（所有交易均在本期內），則全部計入

## 五、觸發指令與執行流程

### 觸發詞 A：`{客戶名} 資產現況`
**範例：** `林峻毅 資產現況`、`靜怡 資產現況`

**執行流程：**

1. 讀取 `~/Desktop/Clients/` 中該客戶最新日期的 Excel（用 `openpyxl data_only=True` 直接讀取，**不使用 xlsx skill**）
2. 從 `Investment Portfolio` 工作表擷取左側持有部位、右側已實現交易、彙總欄位
3. **依第二節結帳週期規則，篩選本期交易**（買入日期 ≥ 本期起算日）
4. 計算交易績效指標（詳見第六節）
5. 產出一頁式 PDF 報告（詳見第七節）
6. 報告存至：`~/Desktop/Clients/{資料夾名稱}/`
7. 圖檔存至：`~/Desktop/Clients/{資料夾名稱}/pic/`

---

### 觸發詞 B：`客戶總覽`

**執行流程：**

1. 讀取本文件第一節客戶清單，取得所有客戶名稱與初始投資額
2. 依第三節路徑規則，逐一找到每位客戶最新 Excel（`openpyxl data_only=True`，**唯讀**）
3. 從 `Investment Portfolio` 工作表讀取：
   - `I25`：期初投資額（本期滾動）
   - `I27`：當前餘額（= AUM）
   - `I29`：本期總損益
4. 計算各客戶指標（詳見 6.4 節）
5. 產出一頁式 PDF 報告（詳見 8.1 節），存至 `~/Desktop/Clients/AUM_Overview/`
   - 檔名：`客戶總覽_{YYYYMMDD}.pdf`
   - 版面與風格完全對齊個人報告（酒紅金暗色主題，同 7.2 節配色）
   - 版面結構：標題列 → KPI 4 卡（總AUM、客戶數、加權本期報酬、加權累計報酬）→ AUM圓餅（左）+ 報酬率雙橫條（右）→ 客戶明細卡（底部5欄）
   - 卡片底色統一用 `#2A1515`，頂部裝飾線顏色依累計報酬正負（綠/紅）
6. 在對話輸出文字彙總表格（詳見 8.1 節）
7. ⚠️ **嚴禁對任何客戶 Excel 進行任何寫入或修改**

## 六、計算邏輯

### 6.1 帳戶層級（來自 Excel 彙總欄位）

| 指標 | 計算方式 |
|------|---------|
| 期初投資額 | I25 |
| 當前餘額 | I27 |
| 本期總損益 | I29 |
| 本期總報酬 | I29 ÷ I25 × 100% |

### 6.2 交易績效（僅計本期交易）

| 指標 | 計算方式 |
|------|---------|
| 總交易次數 | 本期交易筆數 |
| 勝率（依個股） | 同一股票所有交易 PnL 加總後為正 = 贏，計算贏的個股數 ÷ 總個股數 |
| 平均持有天數 | 本期交易 Q 欄平均 |
| 月平均報酬率 | 本期總報酬 ÷ 本期月數 |
| 最佳交易 | 本期 X 欄最大值 |
| 最差交易 | 本期 X 欄最小值 |
| 獲利排行 Top 3 | 本期同股加總後排序前三 |
| 平均報酬（每筆） | 本期 Y 欄平均 × 100% |

### 6.4 全客戶彙總計算（觸發詞 B 專用）

| 指標 | 計算方式 |
|------|---------|
| AUM | 各客戶 I27 |
| AUM 佔比 | 各客戶 I27 ÷ 所有客戶 I27 加總 × 100% |
| 本期報酬率 | I29 ÷ I25 × 100% |
| 累計報酬率 | (I27 ÷ 初始投資額 − 1) × 100% |
| 總 AUM | 所有客戶 I27 加總 |
| 加權平均本期報酬率 | Σ(各客戶 I29) ÷ Σ(各客戶 I25) × 100% |
| 加權平均累計報酬率 | Σ(各客戶 I27 − 初始投資額) ÷ Σ(各客戶初始投資額) × 100% |

> 若某客戶 Excel 找不到，標注「⚠️ 檔案未找到」，不影響其他客戶計算。

---

### 6.3 損益明細（底部 3 卡片）

| 指標 | 計算方式 |
|------|---------|
| 初始投資額 | 第一節表格中的固定值（客戶首次入金金額） |
| 當前餘額 | I27 |
| 累計報酬率 | (I27 ÷ 初始投資額 − 1) × 100% |

## 七、一頁式 PDF 報告規範

### 7.1 檔案命名與路徑
- 檔名：`{客戶名}_{製作日期YYYYMMDD}_資產現況.pdf`
- 路徑：`~/Desktop/Clients/{資料夾名稱}/`
- 例：`林峻毅_20260327_資產現況.pdf`

### 7.2 主題配色（酒紅金 暗色主題）

| 元素 | 色碼 | 用途 |
|------|------|------|
| 背景 | `#1a0a0a` | 全頁底色 |
| 酒紅 | `#8B1A1A` | 標題列 |
| 金色 | `#C8A951` | 標題文字、分隔線、徽章 |
| 淺金 | `#D4BC7A` | 副標題、小節標題 |
| 暖白 | `#F5F0E8` | 正文文字 |
| 卡片底 | `#2A1515` | 資訊卡背景 |
| 卡片邊 | `#4A2525` | 資訊卡邊框 |
| 正值 | `#4CAF50` | 損益正值（綠） |
| 負值 | `#E74C3C` | 損益負值（紅） |
| 灰文 | `#9E8E7E` | 標籤、說明文字 |

### 7.3 版面結構

使用 `matplotlib` + `GridSpec(4, 2, height_ratios=[0.07, 0.055, 0.52, 0.355])`

| 區域 | 內容 |
|------|------|
| **第一列（全寬）** | 酒紅標題列：`{客戶名} 資產現況報告`，右側：`{Fee-Based/Non-Fee} | {日期}` |
| **第二列（全寬）** | 4 張 KPI 卡片：期初投資額、當前餘額、本期總損益、本期總報酬 |
| **第三列左** | 交易績效總覽（詳見 7.4） |
| **第三列右** | 投資報酬結構 + 獲利排行 Top 3 + 交易概況（詳見 7.5） |
| **第四列（全寬）** | 損益明細：3 張大卡片 — 初始投資額、當前餘額、累計報酬率 |
| **頁尾** | `資料截至 {Excel日期} | 製表日期 {今日}` |

### 7.4 第三列左：交易績效總覽

顯示「本期: YYYY/MM ~ YYYY/MM」標注

**4 行統計卡：**
1. 總交易次數（X 筆）
2. 勝率（依個股）（X%（X 檔））— 勝率 ≥50% 綠色，<50% 紅色
3. 平均持有天數（X 天）
4. 月平均報酬率（+X.XX%）— 正綠負紅

**最佳/最差交易：**
- 左卡（深綠底 `#1B3A1B`）：▲ 股票名 / +金額 / (+報酬率%)
- 右卡（深紅底 `#3A1B1B`）：▼ 股票名 / -金額 / (-報酬率%)

### 7.5 第三列右

**投資報酬結構（堆疊長條圖）：**
- 灰色段：期初投資額（I25）
- 綠/紅段：本期損益（I29）
- 右下標注：`本期總報酬: +X.XX%`

**獲利排行 Top 3：**
- 標注「（本期 YYYY/MM ~ YYYY/MM）」
- 金/銀/銅徽章 + 股票名 + 綠色長條 + 金額

**交易概況（4 格迷你卡）：**
- 交易筆數 / 操作個股 / 平均報酬 / 最常交易

### 7.6 第四列：損益明細（3 張大卡片）

| 卡片 | 背景色 | 文字色 |
|------|--------|--------|
| 初始投資額 | `#3A3535`（灰） | 暖白 |
| 當前餘額 | `#1A2A4A`（深藍） | `#6AAFFF` |
| 累計報酬率 | `#1B3A1B` 正 / `#3A1B1B` 負 | 綠/紅 |

卡片尺寸：寬 0.28、高 0.60（axes 座標），金色頂部裝飾線。

### 7.7 技術規範
- **直接用 Python（openpyxl + matplotlib）**，不呼叫 xlsx skill
- 圖檔 DPI：150
- 尺寸：13.5 × 9.5 吋
- 字體：`Heiti TC`（確保中文顯示正常）
- 若有持倉（左側有資料），第三列左改為**圓餅圖**（持倉配置佔比）
