# Screening.md｜月初選股分析規範

> 每月初輸入 40 支股票代號，自動建立新工作表並填入基本資料，加速選股流程。
> 目標：從 ~3 小時壓縮至 ~30 分鐘（人工只需填備註 + 做兩階段判斷）。

---

## 一、主要檔案路徑

| 項目 | 路徑 |
|------|------|
| 正式檔案 | `~/Desktop/Investment/Jeff_Stock Analysis.xlsx` |
| 作業草稿 | `~/Desktop/Investment/Jeff_Stock Analysis_Draft.xlsx` |

> ⚠️ **預設操作 Draft 檔案**，除非使用者明確說「寫入正式檔案」。

## 二、觸發規則

### 觸發詞 A：建立新月工作表
`月初選股 YYYYMM`（附上 40 支股票代號清單）

**範例：**
```
月初選股 202604
2360, 6291, 3526, 1590, ...（40 支代號）
```

### 觸發詞 B：更新月中 / 月末追蹤價格
`更新 YYYYMM 收盤價 M/D`

**範例：**
```
更新 202604 收盤價 4/30
```

### 觸發詞 C：月底績效回顧
`選股回顧 YYYYMM`

## 三、工作表欄位規格

### 3.1 欄位對照表

| 欄 | 欄位名稱 | 來源 | 說明 |
|----|---------|------|------|
| A | 排序 | 使用者輸入順序 | 1-based，手動 |
| B | 公司代號 | 使用者輸入 | 4-digit |
| C | 公司名稱 | **API 自動填入** | `name` |
| D | {前前月末}收盤價 | **TWSE API 自動填入** | 前前月最後交易日（詳見 3.3 節） |
| E | {前月末}收盤價 | **TWSE API 自動填入** | 前月最後交易日（詳見 3.3 節） |
| F | 前個月漲幅 | **公式自動計算** | `=(E-D)/D` |
| G | 排名 | **公式自動計算** | 依 F 欄排名（`RANK` 公式） |
| H | {追蹤日}收盤價 | 本月第一個交易日，月中 / 月末追蹤更新 | 初始為本月第一交易日收盤價 |
| I | 第1個月漲幅 | **公式自動計算** | `=IF(H="","",(H-E)/E)` |
| J | 排名 | **公式自動計算** | 依 I 欄排名 |
| K | 產業（子類） | **API + 人工分類** | 依產品子產業分類（詳見 3.4 節） |
| L | 備註 | **自動填入 + 人工補充** | 近期新聞主要題材（10字內），可手動覆蓋 |
| M | 第一階段選擇 | **自動預篩 + 人工確認** | `考慮` / `否` / 空白（詳見 3.5 節） |
| N | 第二階段選擇 | **人工判斷** | `選擇` / `否` / 空白 |
| O | TTM EPS | **API 自動計算** | `net_income_ttm / share_capital`，保留 2 位小數 |
| P | Current PE | **公式自動計算** | `=IF(AND(E<>"",O<>"",O>0),ROUND(E/O,1),"")` |
| Q | EV/EBITDA | **API 自動填入** | `ev_ebitda`；EBITDA 為負或無值時標注「EBITDA為負」（紅色） |
| R | 股票張數 | **API 自動計算** | `share_capital(億股) × 100,000 = 張數`（在外流通股數） |
| S | 市值（億） | **公式自動計算** | `=IF(E<>"",ROUND(R×E/100000,2),"")` 置中對齊 |

### 3.2 欄位顏色規則

| 類型 | 字色 | 底色 | 說明 |
|------|------|------|------|
| A, B 欄（排序、代號） | 藍色 `#1F4E79` | 無 | 使用者輸入 |
| L 欄（備註） | 黑色 | 淺灰 `#F2F2F2` | 置中，自動填入可手動覆蓋 |
| M 欄（第一階段選擇） | 黑色 | 淺灰 `#F2F2F2` | 置中，自動預篩後人工確認 |
| N 欄（第二階段選擇） | 藍色 `#1F4E79` | 無 | 純人工 |
| API 填入欄（C, D, E, O, Q, R） | 黑色 | 無 | 自動帶入 |
| 公式計算欄（F, G, I, J, P, S） | 黑色 | 無 | 自動計算 |
| K 欄（產業，4支以上群體） | 黑色 | 依子類色 | IC設計/電腦周邊/石化塑膠各自上色 |
| 行 2-41 其他欄 | 黑色 | **無底色** | K/L/M 欄除外 |
| 漲幅正值 | 綠色 `#375623` | — | 條件格式 |
| 漲幅負值 | 紅色 `#C00000` | — | 條件格式 |
| EPS為負（P 欄） | 紅色 `#C00000` | — | 顯示「EPS為負」文字 |
| EBITDA為負（Q 欄） | 紅色 `#C00000` | — | 顯示「EBITDA為負」文字 |
| 第二階段「選擇」列 | — | 黃色 `#FFF2CC` | 整列標示 |

### 3.3 收盤價日期邏輯

> 以建表月份（YYYYMM）為基準，例：202604 = 4 月分析

| 欄位 | 取哪天的收盤價 | 範例（202604）|
|------|--------------|--------------|
| D（前前月末收盤價） | 前前月最後交易日 | 2/26 |
| E（前月末收盤價） | 前月最後交易日 | 3/31 |
| H（追蹤收盤價） | 本月第一個交易日（使用者告知），後續可追蹤更新 | 4/1 |
| Row 44（大盤指數） | 對應 D/E/H 欄日期的加權指數收盤值 | 35,414.49 / 35,095.09 / TBD |

> ⚠️ TWSE API（`STOCK_DAY`）提供歷史收盤價。日期格式：`YYYYMMDD`，ROC 格式如 `115/02/26`。
> 每月初建表時，D 欄（前前月末）和 E 欄（前月末）需從 TWSE API 批次抓取歷史收盤價。

### 3.4 產業子類分類規則

> 使用產品型態分類（非 TWSE 大分類），**只對 4 支以上同產品群體上色**。

| 子產業 | 顏色 | 說明 |
|--------|------|------|
| IC設計 | 淡紫 `#C5B4E3` | 包含面板IC、觸控IC、電源IC等設計公司 |
| 電腦周邊 | 淡藍 `#B0D8F0` | 電腦硬體、周邊設備製造 |
| 石化塑膠 | 淡琥珀 `#F5D6A0` | 石化原料、塑膠製品 |
| 其餘 | 無底色 | 不足 4 支同類的產業，不上色 |

**操作規則：**
- 每月建表時，依股票實際產品重新判斷子產業歸屬
- 歸屬 4 支以上的群體才上色，其餘清空底色
- K 欄文字顯示子產業名稱（非 TWSE 原始分類）

### 3.5 第一階段選擇自動預篩規則

| 條件 | 動作 |
|------|------|
| TTM EPS < 0（虧損股） | 自動填「否」 |
| EV/EBITDA 為「EBITDA為負」 | 自動填「否」 |
| 例外：備註欄含轉虧為盈跡象 | 保留空白，人工判斷 |
| 其餘 | 保留空白，人工判斷 |

> 第二階段（N 欄）全部留空，由人工決定。

## 四、執行流程（觸發詞 A — 月初建表）

### Step 1 — 準備資訊
使用者須提供：
- 分析月份（YYYYMM）
- 40 支股票代號清單
- 本月第一個交易日日期（供 H 欄標題使用）
- 前月最後交易日日期（供 E 欄 TWSE 抓取）
- 前前月最後交易日日期（供 D 欄 TWSE 抓取）

### Step 2 — 建立工作表
1. 將 `~/Desktop/Investment/Jeff_Stock Analysis.xlsx` **複製一份**，存為 `~/Desktop/Investment/Jeff_Stock Analysis_Draft.xlsx`（覆蓋舊版）
2. 開啟 `Jeff_Stock Analysis_Draft.xlsx`，新增工作表，命名 `YYYYMM`（例：`202604`）
3. 複製上月工作表格式（欄寬、字型、無格線）做為新表基礎

### Step 3 — 批次呼叫 API（five91）
一次呼叫 `/api/metrics`，取出所有 40 支：
```
需要欄位：name, category, net_income_ttm, share_capital, ev_ebitda, market_cap
```
計算：
- TTM EPS = `net_income_ttm / share_capital`（保留 2 位）
- EV/EBITDA = `ev_ebitda`（無值或負 EBITDA 標注「EBITDA為負」）
- 張數 = `share_capital × 100,000`

### Step 4 — TWSE 批次抓取歷史收盤價
使用 TWSE API `STOCK_DAY` 分別抓取 D 欄（前前月末）和 E 欄（前月末）收盤價：
```
GET https://www.twse.com.tw/rwd/zh/afterTrading/STOCK_DAY?stockNo={代號}&date={YYYYMMDD}&response=json
```
- 找對應 ROC 日期列（如 `115/02/26`），取第 7 欄（收盤價）
- 上市股：TWSE；找不到時嘗試 OTC（tpex.org.tw）
- 找不到的股票：D/E 欄留空，最終回報缺失清單

### Step 5 — 填入工作表
| 欄 | 內容 |
|----|------|
| A | 排序（1-40） |
| B | 代號 |
| C | 公司名稱（API） |
| D | 前前月末收盤價（TWSE） |
| E | 前月末收盤價（TWSE，建表時為空，待填） |
| F | 公式 `=(E2-D2)/D2` |
| G | 公式 `=RANK(F2,$F$2:$F$41,0)` |
| H | 留空（H1 標題 = 本月第一交易日，如 `4/1收盤價`） |
| I | 公式 `=IF(H2="","",(H2-E2)/E2)` |
| J | 公式 `=IF(I2="","",RANK(I2,$I$2:$I$41,0))` |
| K | 子產業分類（4支以上上色） |
| L | 近期新聞題材（WebSearch，10字內）|
| M | 自動預篩「否」（EPS<0 或 EBITDA為負，例外保留空白） |
| N | 空白 |
| O | TTM EPS（`net_income_ttm/share_capital`，格式 `0.00`） |
| P | 公式 `=IF(AND(E<>"",O<>"",O>0),ROUND(E/O,1),"")` |
| Q | EV/EBITDA（API，無值→「EBITDA為負」紅字） |
| R | 張數（`share_capital×100000`，格式 `#,##0`） |
| S | 公式 `=IF(E<>"",ROUND(R*E/100000,2),"")` 置中 |

### Step 6 — Row 44 大盤指數
| 欄 | 內容 |
|----|------|
| D44 | 前前月末加權指數（TWSE MI_5MINS_HIST） |
| E44 | 前月末加權指數（建表時留空，待填） |
| H44 | 本月追蹤日加權指數（留空） |

### Step 7 — 回報
```
✅ YYYYMM 工作表建立完成

成功帶入：XX / 40 支
⚠️ 找不到資料：XXXX（原因）

建議下一步：
1. 確認 L 欄備註（已自動填入新聞題材，可手動補充）
2. 確認 M 欄第一階段（EPS<0 / EBITDA為負已自動填「否」）
3. E 欄（前月末收盤價）將於月末收盤後更新
```

## 五、執行流程（觸發詞 B）

### 更新月中 / 月末追蹤價格

1. 開啟對應月份工作表
2. 從 TWSE API 或 five91 查詢指定日期收盤價
3. 寫入 H 欄（更新 H1 標題為指定日期）
4. 同步更新 H44 當日加權指數
5. I、J 欄公式自動重算
6. 回報漲幅前 5 / 後 5 排名摘要

## 六、執行流程（觸發詞 C）

### 月底績效回顧

1. 讀取指定月份工作表
2. 計算：
   - 第一階段「考慮」→ 最終入選率
   - 「選擇」股票 vs 全體平均 第1個月漲幅比較
   - 最佳 / 最差 3 支
3. 在對話輸出回顧摘要（不寫入 Excel）

## 七、API 說明

### five91.onrender.com（主要財務資料）

```
GET https://five91.onrender.com/api/metrics
回傳 JSON 結構：{"stocks": {"1101": {...}, ...}}
```

**常用欄位：**
- `name`：公司名稱
- `category`：TWSE 產業分類（參考用，K 欄用子產業覆蓋）
- `net_income_ttm`：近 12 個月淨利（億元）
- `share_capital`：在外流通股數（億股）
- `ev_ebitda`：EV/EBITDA 倍數
- `market_cap`：市值（億元）
- `price`：當前股價（僅參考，歷史價格用 TWSE API）

**計算公式：**
- TTM EPS = `net_income_ttm / share_capital`（億元 ÷ 億股 = 元/股）
- 張數 = `share_capital × 100,000`（億股 → 張）

### TWSE API（歷史收盤價）

```
GET https://www.twse.com.tw/rwd/zh/afterTrading/STOCK_DAY?stockNo={代號}&date={YYYYMMDD}&response=json
取 data 陣列中 ROC 日期 == 目標日 的第 7 欄（index 6）
```

### TWSE 加權指數 API

```
GET https://www.twse.com.tw/rwd/zh/TAIEX/MI_5MINS_HIST?date={YYYYMMDD}&response=json
取 data 陣列中 ROC 日期 == 目標日 的第 5 欄（index 4，收盤指數）
```

## 八、選股輔助提示（每次建表後自動輸出）

```
第一階段「考慮」篩選條件：
  ✅ 前月漲幅排名前 30%（排名 ≤ 12）
  ✅ 有明確題材（備註欄有內容）
  ✅ PE < 30x（排除高估值）
  ❌ 虧損股（TTM EPS < 0）→ 已自動填「否」
  ❌ EBITDA 為負 → 已自動填「否」

第二階段「選擇」篩選條件：
  ✅ 第一階段考慮 + PE 10-25x
  ✅ 有催化劑（題材 / 儲董看好 / 法說會）
  ✅ 市值 < 500 億（流動性適中的中小型股）
  目標選出 3-6 支
```

- 預設操作 `Jeff_Stock Analysis_Draft.xlsx`，確認結果正確後再說「寫入正式檔案」
