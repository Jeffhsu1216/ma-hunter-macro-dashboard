# SwingTrade.md｜短線交易分析規範

> 系統化追蹤短線交易（1-4 週持有），建立交易紀錄，定期回顧勝率與風險報酬比。
> 目標：將內線研判轉化為可量化的決策框架，80% 系統化篩選 + 20% 主觀判斷。

---

## 一、核心理念

**80/20 原則：**
- **80% 系統化**：進場條件清單、停損停利規則、部位控管、週線技術面篩選
- **20% 主觀判斷**：消息面品質評估、時機微調、加減碼判斷

**交易紀律：**
- 每筆交易進場前必須填寫「進場理由」與「預設停損」
- 不符合進場條件清單的交易 → 標記為「紀律外」，納入勝率分析但獨立計算
- 週五收盤後固定回顧本週持倉

## 四、交易紀錄欄位規格（SwingTrade_Log.xlsx）

### 4.1 工作表：`交易紀錄`

| 欄 | 欄位名稱 | 說明 |
|----|---------|------|
| A | 編號 | 自動遞增（ST-001, ST-002, ...） |
| B | 進場日期 | YYYY/MM/DD |
| C | 代號 | 4-digit |
| D | 名稱 | API 帶入 |
| E | 產業 | 子產業分類 |
| F | 進場價 | 使用者提供 |
| G | 張數 | 使用者提供 |
| H | 進場金額 | 公式 `=F×G×1000` |
| I | 消息來源 | `內線` / `技術面` / `題材` / `法說會` / `其他` |
| J | 進場理由 | 使用者簡述或自動帶入（50字內） |
| K | 預設停損價 | 使用者設定或建議值（預設 -7%） |
| L | 預設停利價 | 使用者設定或建議值（預設 +15%） |
| M | 風險報酬比 | 公式 `=(L-F)/(F-K)` |
| N | 出場日期 | 出場時填入 |
| O | 出場價 | 出場時填入 |
| P | 報酬率 | 公式 `=(O-F)/F` |
| Q | 損益金額 | 公式 `=(O-F)×G×1000` |
| R | 持有天數 | 公式 `=N-B` |
| S | 結果 | 公式：報酬率 > 0 →「✓ 獲利」、< 0 →「✗ 虧損」 |
| T | 是否符合紀律 | `是` / `否`（進場時是否符合 80% 系統條件） |
| U | 事後檢討 | 出場後手動填寫（50字內） |

### 4.2 工作表：`績效儀表板`

| 區域 | 內容 |
|------|------|
| A1:D6 | 總覽 KPI：總交易數、勝率、平均報酬率、獲利因子（Profit Factor） |
| A8:D15 | 紀律內 vs 紀律外勝率對比 |
| A17:D25 | 依消息來源分組勝率 |
| F1:L15 | 累積損益曲線圖 |
| F17:L30 | 月度勝率趨勢圖 |

## 五、短線分析流程（觸發詞 C）

> 收到「短線分析 {代號}」時，自動執行以下評估：

### Step 1 — 基本資料
從 `five91.onrender.com/api/metrics` 取得：
- 公司名稱、產業、股價、TTM EPS、EV/EBITDA、市值

### Step 2 — 技術面快篩（週線）
從 TWSE API 取得近 60 個交易日收盤價，計算：

| 指標 | 計算方式 | 短線意義 |
|------|---------|---------|
| 5MA / 20MA / 60MA | 簡單移動平均 | 趨勢方向 |
| 5MA 是否 > 20MA | 黃金交叉判斷 | 短期動能轉強 |
| 現價 vs 20MA 偏離度 | `(Price - MA20) / MA20` | 超買超賣參考 |
| 近 5 日漲幅 | `(今收 - 5日前收) / 5日前收` | 短期動能 |
| 近 20 日波動率 | 20日報酬率標準差 × √252 | 風險評估 |
| 成交量趨勢 | 5日均量 / 20日均量 | 量能確認 |
| 近 3 月日均量 | 近 60 日成交量平均（張） | 流動性評估 |

### Step 2.5 — 籌碼面快篩

從 TWSE T86 API 取得近 20 個交易日三大法人買賣超：

| 指標 | 計算方式 | 短線意義 |
|------|---------|---------|
| 外資 20 日累計淨買超 | Σ(每日外資淨買賣) | 外資動向，正=站買方 |
| 外資連續買/賣天數 | 連續淨買或淨賣天數 | 趨勢持續性 |
| 投信 20 日累計淨買超 | Σ(每日投信淨買賣) | 投信佈局訊號 |

### Step 3 — 消息面評估
WebSearch 查詢近 7 天新聞：
- 利多/利空分類
- 主要題材摘要（30字內）
- 是否有法人買賣超、董監持股異動

### Step 4 — 進場條件清單（80% 系統化）

> 至少滿足 5/7 項才建議進場：

| # | 條件 | 說明 |
|---|------|------|
| 1 | TTM EPS > 0 | 排除虧損股（短線也不碰） |
| 2 | 5MA > 20MA | 短期趨勢向上 |
| 3 | 現價在 20MA 上方 ≤ 10% | 未過度偏離，仍有空間 |
| 4 | 近 3 個月日均量 > 1,000 張 | 流動性足夠，查 TWSE 近 60 日實際成交量驗證 |
| 5 | 近 20 日外資淨買超 > 0 | 外資站買方；若連續大幅賣超則為警訊 |
| 6 | 最近月營收 YoY > 0% | 基本面動能仍在，排除衰退中標的 |
| 7 | 有明確催化劑 | 新聞 / 內線消息面支持 |

### Step 5 — 輸出評估報告

```
📊 短線分析：{代號} {名稱}
━━━━━━━━━━━━━━━━━━━━

基本面：TTM EPS {X} | PE {X}x | EV/EBITDA {X}x | 市值 {X}億
營收動能：最近月營收 {X}億（YoY {+/-X}%）
技術面：5MA{↑/↓}20MA | 偏離 20MA {X}% | 5日漲幅 {X}%
量能：5日均量/20日均量 = {X}（{放量/縮量/持平}）| 3月日均量 {X}張
籌碼面：外資 20日淨買超 {X}張 | 連續{買/賣}{X}日
消息面：{摘要}

進場條件：{X}/7 ✓
建議：{建議進場 / 觀望 / 不建議}
建議停損：{price}（-7%） | 建議停利：{price}（+15%）
風險報酬比：{X}:1
```

## 六、進場記錄流程（觸發詞 A）

1. 開啟 `SwingTrade_Log.xlsx`，`交易紀錄` 工作表
2. 在最末列新增一行
3. 填入：編號、進場日期（今天）、代號、名稱（API）、產業、進場價、張數
4. 自動計算：進場金額、預設停損（-7%）、預設停利（+15%）、風險報酬比
5. 詢問使用者：消息來源、進場理由（或自動帶入短線分析結果）
6. 詢問是否調整停損停利
7. 回報記錄完成

---

## 七、出場記錄流程（觸發詞 B）

1. 開啟 `SwingTrade_Log.xlsx`，找到對應代號的未出場紀錄
2. 填入：出場日期（今天）、出場價
3. 公式自動計算：報酬率、損益金額、持有天數、結果
4. 提示使用者填寫「事後檢討」
5. 回報損益摘要

## 九、績效總覽流程（觸發詞 E）

1. 讀取全部已出場交易
2. 產出 PDF 報告至 `~/Desktop/Investment/Reports/SwingTrade/`
3. 包含：
   - 累積損益曲線
   - 月度勝率長條圖
   - 消息來源勝率比較（內線 vs 技術面 vs 題材）
   - 紀律內 vs 紀律外勝率比較
   - 持有天數分佈圖
   - 最佳 / 最差 5 筆交易
4. 酒紅金暗色主題（與客戶報告一致）

## 十、風險控管規則

| 規則 | 說明 |
|------|------|
| 單筆停損上限 | -7%（可依個股波動調整至 -10%） |
| 單筆最大部位 | 總資金的 20% |
| 同時持有上限 | 5 檔 |
| 單週最大虧損 | 達總資金 -5% → 暫停交易至下週 |
| 連續虧損 | 連 3 筆虧損 → 降低部位至 50% |
